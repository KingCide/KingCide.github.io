用Claude Code给博客加碎碎念页面

发表于 2026-02-11 更新于 2026-02-19 分类于 学习印记 阅读次数：
使用 Claude Code AI 工具为 Hexo 博客添加碎碎念（短想法）页面的开发过程记录。

一直想在博客里加一个随手记录想法的地方，不用写成完整的文章，也不需要标题和分类，就是随便说两句。作为一个简单项目，测试下claude code pro 能有多少opus的用量。

需求

有时候只是一句感想、一个灵感，不足以专门开一篇 post 。我想要的是一个类似微博/说说的轻量页面：

  • 纯静态，不依赖后端，跟博客一起部署到 GitHub Pages
  • 数据用 JSON 文件存在本地，方便 git 管理
  • 独立页面，不经过 Hexo 主题渲染
  • 命令行一行命令就能添加新想法
  • TODO：支持iphone移动端快速稳定流程上传

macOS 快捷发布

命令行方式有几个痛点：每次要 cd 到项目目录，打开终端，而且终端粘贴中文偶尔会乱码。最初用 AppleScript 的 display dialog 做了一版，但它的输入框是单行的，按回车直接触发按钮，没法写多行内容。

最终方案是用 JXA（JavaScript for Automation）+ Cocoa 的 NSTextView，做成一个原生多行编辑器：

交互流程是：启动 → 选择「新建想法」/「编辑已有」/「删除」。新建弹出多行编辑器，回车就是换行；编辑先选条目再打开编辑器，内容预填好；删除有二次确认。

后续

目前这个方案只能在电脑上添加想法。如果以后想在手机上也能发，可以：

  • 把 data.json 的读写迁移到 GitHub API 或 Cloudflare Workers
  • app.js 改为从 API 获取数据
  • 页面本身不需要改动

之后还可以让它生成更多的主题
