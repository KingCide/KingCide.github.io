博客 SEO 优化实践：从零开始让 Google 和百度收录你的 Hexo 博客

发表于 2026-02-19 分类于 学习印记 阅读次数：
记录个人 Hexo 博客在 Google 和百度搜索引擎的 SEO 优化全过程，包括 URL 主动推送配置（hexo-url-submission）、Google Indexing API 接入、front-matter 优化等实践经验。

这个博客运行了好几年，但一直是写给自己看的，从来没有考虑过搜索引擎优化。最近把站点提交到 Google 和百度之后发现，不做 SEO 优化的博客，基本等于在互联网上隐身。于是开始了一轮系统性的优化，本文记录整个过程。

主动推送 URL 到搜索引擎

最基本的一步：让搜索引擎知道你的页面存在。除了被动等待爬虫发现，还可以通过 API 主动告诉搜索引擎我们更新了这些页面。

hexo-url-submission 插件

hexo-url-submission 是一个 Hexo 插件，可以在 hexo deploy 时自动将新页面的 URL 批量推送给 Google、Bing、百度和神马搜索。一条命令完成部署 + 推送，非常省事。

踩过的坑

  1. **prefix 默认值是 ['post']**：这个插件的默认配置假设你的文章 URL 以 /post 开头，但我的 permalink 格式是 :title/，导致所有 URL 都被过滤掉了。设置 prefix: [] 就能匹配所有路径。

  2. type: ‘latest’ 的判断逻辑：latest 模式通过比较源文件的 mtime（修改时间）和当天日期来判断是否需要提交。如果你 hexo clean 后重新生成，源文件的修改时间不会变，所以可能没有任何 URL 被选中。首次推送建议用 type: 'all'，之后再改回 latest。

  3. 百度每日配额：百度站长平台默认每天只能推送 10 条 URL（新站额度很低），超出会返回 400 over quota。合理设置 count 避免浪费请求。

Google Indexing API 配置

Google 的推送不是简单的 token，而是需要一个 Service Account（服务账号）的 JSON 密钥文件。步骤稍微复杂：

  1. 创建项目 & 启用 API：打开 Google API Console，创建项目并启用 Indexing API。

  2. 创建服务账号：在 Service Accounts 页面创建服务账号，下载 JSON 密钥文件，重命名为 google.json 放到博客根目录。

  3. 授权服务账号：在 Google Search Console 中，将 JSON 文件里 client_email 字段的邮箱添加为站点所有者。

  4. 安全：在 .gitignore 中添加 google.json，防止密钥泄露到 GitHub。
