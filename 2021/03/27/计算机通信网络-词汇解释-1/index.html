计算机通信网络-词汇解释-1

发表于 2021-03-27 更新于 2021-03-30 分类于 学习印记 阅读次数：

单元一词汇解释

核心词汇

OSI

开放式系统互联通信参考模型（Open System Interconnection Reference Model）将计算机网络体系结构划分为以下七层：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。

它由ISO提出，目的是提供一个使计算机在全球范围内互联为网络的标准框架。

它并未提供一个可以实现的方法，而是一个制定标准时所使用的概念性框架。

封装

封装（Encapsulation）是一种通讯协定的设计方法——将网络功能抽象出来，仅对外公开接口，而对高层隐藏底层功能。

封装使网络通信实现模块化并易于管理，使数据包在不同设备之间传输时可以准确可靠地发送到目的地。

封装的过程就是在分层模型的每一层中，添加协议头和尾到对应的协议数据单元（PDU）中。

协议

协议（protocol）指的是通信计算机双方必须共同遵从的一组约定，包括信息的格式以及如何发送和接受信息的一套规则。

所有接入网络的计算机若想进行信息交换，就需要确定的规则保证在不同的物理设备间也可互相理解，此时就需要协议配合封装来实现。

协议的三要素为：语法、语义、时序。语法——即数据与控制信息的结构或格式；语义——即需要发出何种控制信息，完成何种动作以及做出何种响应；时序——即事件实现顺序的详细说明。

实体

实体（Entity）指的是层中的活跃元素。

这个称呼是为了强调网络中发挥作用的东西的存在性。

它可以是真实的硬件也可以是虚拟的软件，同时每层中可以同时存在多个实体。

时序图

时序图指以时间先后关系描述通信交互过程的图。

它可以清晰地以时间顺序展示两方交流的过程。

一个时序图需要包括以下内容：时间轴、对等层、对等层实体间时间、上下层实体间接口以及上下层实体间事件。

一般词汇

IETF

国际互联网工程任务组（Internet Engineering Task Force）是一个由为互联网技术工程及发展做出贡献的专家自发参与和管理的国际民间机构。

主要任务是负责互联网相关技术标准的研发和制定。

IETF制定了很多重要的网络底层协议，例如TCP协议、IP协议、HTTP协议。它的身上充分体现了自由、开放、合作、共享的精神。

ITU-T

国际电信联盟电信标准分局(ITU Telecommunication Standardization Sector)是国际电信联盟管理下的专门制定电信标准的分支机构。

在客户和服务商对通讯技术的要求急剧增加时，需要这样一个机构加快标准制定的进程来适应这一需求。

ITU-T的国际标准通常被称为建议，且协议不是开放的，一般不提供免费下载。

IEEE

电气与电子工程师协会（Institute of Electrical and Electronics Engineers）是一个国际性的电子技术与信息科学工程师的协会，也是目前全球最大的非营利性专业技术学会。

它致力于电气、电子、计算机工程和与科学有关的领域的开发和研究。

IEEE制定了全世界电子和电气还有计算机科学领域30%的文献，另外它还制定了超过900个现行工业标准。每年它还发起或者合作举办超过300次国际技术会议。

RFC

请求意见稿（Request for Comments）是由互联网工程任务组（IETF）发布的一系列备忘录。文件收集了互联网相关信息，以编号排定。

它目的是建立一种快速共享Internet网络研究思想的方式，是Internet开放性的产物。

文件在成为官方标准之前，一般经历这几个阶段：因特网草案、建议标准、草案标准、因特网标准。
