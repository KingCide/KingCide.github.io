hexo及next更新

发表于 2021-02-04 更新于 2021-02-05 分类于 学习印记 阅读次数：

这个网站几乎恰好是一年之前建好的，在这一年里未曾维护，内核和主题都已经更新换代，我也要紧跟时势。

Hexo更新

首先，我需要把Hexo由3.x更新到5.3.0

  1. 在博客所在文件夹 git bash，npm install hexo-cli -g，(全局升级）应该也可以在任意文件夹

  2. 接下来按照高人所述

  3. 此时hexo -v 得到了正确的版本，也发现只有在博客的文件夹里，才会显示hexo的版本号。发现Git Bash是基于cmd的，可以执行Linux命令的命令行工具。

  4. 然而我的其他各方面的版本都落后于网友。

  5. 更新之后，首先在hexo clean或hexo s时会报错，WARN Deprecated config detected: "use_date_for_updated" is deprecated, please use "updated_option" instead. See https://hexo.io/docs/configuration for more details.只需要在配置文件中更改这个被替换的名称。

  6. 然后生成的网页仍是乱码，更改了一下主题为landscape之后可以正常显示，所以应该是新版本hexo与老next不兼容的问题。

Next更新

  1. 首先到github仓库直接下载主题的压缩文件，解压到themes文件夹。

    也可以按更新文档使用$ git clone https://github.com/theme-next/hexo-theme-next themes/next-reloaded

  2. 按照说明更改语言，zh-Hans 改名为 zh-CN

  3. 我把之前next的_config.yml复制过来，网站图标文件也复制过来，生成网页时还是报错，说什么TypeError: Cannot read property 'enable' of undefined

  4. 于是决定在新文件里重新配置一遍，发现主题使用的图标由envelope变为fa fa-envelope，用之前的就将没有图标。

  5. 在设置背景时也有问题，由于next主题更新至7.0+版本后取消了 _custom文件夹以及custom.styl，导致新版本不兼容，不过配置文件里有说明，让我到这里去找，终于正常显示背景了。

  6. 最后，成功将新版本部署。
