博客的诞生

发表于 2020-02-02 更新于 2021-02-05 分类于 学习印记 阅读次数：

起源

哥哥叫我学习 github，一方面是为了获取关于各种项目的知识，另一方面我想借助它建立一个个性化的博客。很早以前我便想在博客上写一些东西，不是为了给别人看，就是希望能记录一下学习的过程，以及过程中的感悟，表达自己对电影、音乐的感动之情。因此，我看着视频教程，花了足足三天时间才让这个博客初具雏形。现在我要将孕育的过程梳理一下。

过程

1.node.js

我首先下载了这个程序，后来我开始寻找为什么要下载它，渐渐了解Node.js究竟是什么。官方宣称 “旨在提供一种简单的构建可伸缩网络程序的方法”。它让网站只需要较简单的逻辑便可以满足巨大的流量。

而对于博客来说，我需要用到它内置的npm——Node Package Manager。一个远程代码仓库中存放所有需要被共享的js代码，用户引用对应标识就可以下载对应文件，因此共享代码变得更加方便。

然后我就用npm安装Hexo $ npm install hexo 。

之后再执行这些命令，Hexo会在指定的文件夹中新建文件。

2.Git

关于Git，我在Pro Git上找到了系统的答案。它是一个分布式版本控制系统，客户端可以将代码仓库完整地镜像下来，用于管理和维护代码。

安装完之后，右键就有了Git Bash Here。

3.GitHub

首先我解决了在某些网络下无法连接到github网站的问题（总是用手机的电信热点太麻烦了），经过百度我在HOSTS文件中添加了一段ip就可以上了。应该是本地DNS无法解析导致的。

SSH key（Secure Shell）提供了一种与GitHub通信的方式，通过这种方式，能够在不输入密码的情况下，将GitHub作为自己的remote端服务器，进行版本控制。

所谓”公钥登录”，原理很简单，就是用户将自己的公钥储存在远程主机上。登录的时候，远程主机会向用户发送一段随机字符串，用户用自己的私钥加密后，再发回来。远程主机用事先储存的公钥进行解密，如果成功，就证明用户是可信的，直接允许登录，不再要求密码。

gitbash 中输入 $ ssh-keygen -t rsa -C "1659057796@qq.com" ，在文件 id_rsa.pub 中便是我的公钥，复制到github上即可。

4.Hexo

Hexo 是一个快速、简洁且高效的博客框架。

  • the document>npm install hexo-cli -g
  • hexo init KingCide
  • npm install
  • hexo s -p 5555 到 localhost:5555就能看到了
5.Sublime Text 3

一款用于代码、标记和散文的精致文本编辑器。它的界面与速度令人印象深刻，它还有着丰富的插件从而有强大的功能，然而我现在并不是十分需要。现在它是被用来修改hexo和主题配置的。

跟着教程修改了一些之后，还要安装插件$ npm install hexo-deployer-git --save就差不多可以了。

6.Typora

这真的是一款优雅的 Mrakdown 文本编辑器。它界面简洁优美，使用时体贴舒适，让人爱不释手。不仅能满足我想到的所有需求，更总是让我眼前一亮。以后可以看看Typora 完全使用详解。

终于

我写完了我的第一篇博客，在这个过程中我收益良多。
