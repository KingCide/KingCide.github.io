单元四词汇解释

是什么？为什么？怎么样？

核心词汇

存储转发

存储转发技术是包括报文交换技术和分组交换技术的传统转发技术。

目的是进行错误检测，支持不同速度的输入/输出端口间的交换。

以太网交换机的控制器先将输入端口到来的数据包缓存起来，先检查数据包是否正确，并过滤掉冲突包错误。确定包正确后，取出目的地址，通过查找表找到想要发送的输出端口地址，然后将该包发送出去。

逐站寻径

逐站寻径是分组交换技术的一个过程。

这样做是因为没有事先建立好源到目的的路径，为了达到理论上对资源的最大利用。

在每个交换节点，根据分组中的目的地址，查找路由表，找到出口，实现转发。

洪泛

泛洪（Flooding）是交换机和网桥使用的一种数据流传递技术。

目的是建立MAC地址表。

如数据帧中的目的MAC地址不在MAC地址表中，将从某个接口收到的数据流向除该接口之外的所有接口发送出去。

链路状态（路由算法）

链路状态路由选择协议又称为最短路径优先协议或分布式数据库协议。

为什么：

它基于Dijkstra的最短路径优先算法，从网络或者网络的限定区域内的所有其他路由器处收集信息，最终每个链路状态路由器上都有一个相同的有关网络的信息。

距离矢量（路由算法）

距离矢量是另一种路由算法。

为什么：

距离矢量算法一般用Ford－Fulkerson算法，根据目的地的远近来决定最好的路径。

分组交换

指的是在通信过程中，通信双方以分组为单位、使用存储-转发机制实现数据交互的通信方式。

使两个不同数据率的站之间能够交换分组、链路利用率高……

本质就是存储转发，它将所接受的分组暂时存储下来，在目的方向路由上排队，当它可以发送信息时，再将信息发送到相应的路由上，完成转发。

隧道技术

隧道技术（Tunneling）是一种通过使用互联网络的基础设施在网络之间建立一条虚拟链路以传递数据的方式。

它是虚拟专用网络（Virtual Private Network, VPN）以及移动IP等实现的技术基础

隧道将其他协议的PDU重新封装后通过网络发送，新的PDU提供路由信息，以便通过互联网传递被封装的数据。

网络拥塞

指在分组交换网络中传送分组的数目太多时，网络传输性能下降的情况。

由于存储转发节点的资源——存储空间、带宽容量、处理器性能有限。

通过各种拥塞控制算法解决拥塞。

一般词汇

路径度量（Metric）

metric是指路由算法用以确定到达目的地的最佳路径的计量标准。

通过基于多个metric选择路由，并把它们结合成一个复合的metric，可以确定最佳路径。

常用的metric如下：路径长度、可靠性、延迟、带宽、负载

IP地址子网掩码

是一种用来指明一个IP地址的哪些位标识的是主机所在的子网，以及哪些位标识的是主机的位掩码。

目的是将某个IP地址划分成网络地址和主机地址两部分。

计算方法：

IP路由表最长适配

是CISCO IOS路由器默认的路由查找方式。

为什么：

当路由器收到一个IP数据包时，会将数据包的目的IP地址与自己本地路由表中的表项进行逐位查找，直到找到匹配度最长的条目。

SDN

软件定义网络是网络虚拟化的一种方式。

无须依赖底层网络设备，屏蔽了底层网络设备的差异。而控制权是完全开放的，用户可以自定义任何想实现的网络路由和传输规则策略，从而更加灵活和智能。

通过软件编程的形式定义和控制网络，SDN的本质特点是控制平面和数据平面的分离以及开放可编程性。

MAC层概述

广播式信道

  • 光纤+光柱
  • 双绞线+集线器
  • 同轴电缆+T型头
  • 环网

遇到的问题

  • 两个以上站点使用共享信道
  • 不能有多个站点同时发送
  • 接收站识别送给自己的数据

问题解决目标

  • 低成本——局部范围网络、分布式、无中心
  • 能降低冲突或无冲突
  • 尽量提高信道利用率

共享信道技术

  • 分路复用技术 xDM
  • 分路多路访问（多址）技术 xDMA
  • 多路访问技术 xMA

竞争式访问协议

ALOHA协议

  1. 随时发送数据，等待接收方确认=对发送没有控制，用停等协议解决差错。

  2. 冲突现象严重。

  3. 时槽式ALOHA

    • 将信道使用时间分成离散的时槽。
    • 冲突要么完全重叠，要么完全分开。

CSMA协议

  1. 综述
    • 载波侦听多路访问（Carrier Sense Multiple Access）
    • 发送前侦听线路，空闲则立即发送，若信道忙则监听直到信道空闲
    • CSMA冲突窗口：$ T_p=\frac{d}{2/3c} $
  2. 1坚持CSMA：信道由忙转闲时，以概率1发送数据
  3. P坚持CSMA：信道由忙转闲时，以P概率发送，(1-P)概率随机延时后再尝试发送
  4. 0坚持CSMA：信道由忙转闲时，以0概率发送，都需要随机延时后再尝试发送

CSMA/CD协议

  1. 载波侦听多路访问/冲突检测
  2. 冲突检测时间
  3. 随机延时算法：截断二进制指数回退算法
    • 最小时间片是最大冲突检测时间
    • 根据尝试重传的次数变化延时的范围（二进制指数增加，$2^{10} $后不再增加）

CSMA/CA协议

  1. 无线局域网技术
  2. 冲突域不固定
  3. 通信问题
    • 隐藏站点：RTS（Require to Send）+CTS（Clear to Send）
    • 暴露站点

非竞争式协议

无冲突协议

  1. 预定协议——信道申请
  2. 自定方式——二进制倒计数法
  3. 用特定信息指挥发送——令牌环

以太网

采用了CSMA/CD技术的局域网

共享式以太网

  1. 物理层接口：
    • BNC——同轴电缆接口
    • RJ45——双绞线接口
    • SC——光纤接口
  2. 主要采用曼彻斯特编码
  3. 媒体访问技术——CSMA/CD
  4. 最小帧长64字节，最大帧长1518字节，最多连续冲突次数16次
  5. 以太网编制：MAC地址6字节——单播组播广播

网桥及交换式以太网

  1. 在隔离不同局域网之间的流量的基础上实现联通
  2. 核心技术
    • 存储转发
    • 根据帧内目的地址转发
    • 互联不同LAN协议转换技术
  3. 常用技术
    • 透明网桥
    • 源路由网桥——源端指明路由路径
    • 远程网桥
  4. 交换机——多端口桥
  5. 交换式以太网

虚拟以太网

  1. 软件上划分VLAN
  2. 按端口、MAC地址、用户分
  3. VLAN协议添加头部，特殊的Trunk接口

其他局域网

  1. 无线局域网（WLAN WIFI）IEEE802.11系列协议
  2. 固定宽带WiMax
  3. 蓝牙

主题

利用分层模型描述供应链系统

系统内容

供应链是指产品生产和流通过程中所涉及的原材料供应商、制造商、分销商、零售商以及最终消费者等成员通过与上游、下游成员的连接组成的网络结构。

需求

本系统的需求主要包括两类：由消费者向下的信息传递需求、由供应商向上的商品流通需求。

系统运作过程

从消费者的使用需求出发，将自己的需求信息传递给零售商，进而传递给分销商。

根据需求，此时分销商可能改变商品数量分配或价格，直接将改变返回零售商。

或者分销商继续将生产需求传递给制造商，制造商向供应商传递原材料需求，同时改变生产方式。

供应商得到原料需求后，采购原料后通过物流发送给制造商。

制造商生产出商品后，通过同一个网络将商品一层层地运送到消费者手中。

分层结构

1、供应商：指给制造商提供原材料或零、部件的企业。

2、制造商：负责产品生产、开发和售后服务等。

3、分销企业：为实现将产品送到经营地理范围每一角落而设的产品流通代理企业，相当于网络层路由器。

4、零售企业：将产品销售给消费者的企业，相当于网络层子网路由器。

5、消费者：作为商品流向的最高层，同时负责向下流供应链提供需求信息，相当于应用层。

6、物流企业：即上述层次之外专门提供物流服务的企业，相当于网络的链路。

计算机通信网络-词汇解释-3

发表于 2021-04-25 更新于 2021-05-15 分类于 学习印记

单元三词汇解释

是什么？为什么？怎么样？

核心词汇

多路访问信道

多路访问信道也指广播链路或者随机访问信道。

由于所有结点都能传输帧，就会产生多方竞争信道使用权的问题。

因此，为了解决信道分配的这个问题，产生了静态和动态分配以及各种多路访问协议。

多路复用技术

多路复用技术是把多个低速信道信道组合成一个高速信道的技术。

它可以有效的提高数据链路的利用率，从而使得一条高速的主干链路同时为多条低速的接入链路提供服务。

路复用技术的实质是，将一个区域的多个用户数据通过发送多路复用器进行汇集，然后将汇集后的数据通过一个物理线路进行传送，接收多路复用器再对数据进行分离，分发到多个用户。多路复用通常分为频分多路复用、时分多路复用、波分多路复用、码分多址和空分多址。

竞争式媒体访问控制

介质或媒体访问控制(medium access control)简称MAC。

是解决当局域网中共用信道的使用产生竞争时，如何分配信道的使用权问题。

它定义了数据帧怎样在介质上进行传输。在此处定义了物理寻址、逻辑拓扑、线路控制、帧的传递顺序……

CSMA/CD

载波侦听多路访问/冲突检测（Carrier Sense Multiple Access with Collision Detection），是广播型信道中采用一种随机访问技术的竞争型访问方法。

与其他技术相比，它可以对信道有较高的利用率，在检测到冲突后可以尽快恢复……

在发送之前先监听信道。如果介质空闲，则马上传输。如果介质正在忙，则一直监听到信道空闲，立刻传输。如果检测到冲突，那么立刻停止传输，等待一个随机的时间，之后再重复上面的步骤。

冲突（碰撞）

冲突（Collision）或者说碰撞，意思是若网上有两个或两个以上工作站同时发送数据，在总线上就会产生信号的混合，这样哪个工作站都辨别不出真正的数据是什么。

产生碰撞的原因就是多个信号共用信道。

因此，为了解决信号冲突，产生了静态和动态分配以及各种多路访问协议。

交换机（L2交换机）

交换机是一种用于电（光）信号转发的网络设备。

它可以为接入交换机的任意两个网络节点提供独享的电信号通路。

L2交换机即二层交换机，L2交换机的端口收到任何Frame，就把这个Frame里面的Source MAC Address记录下来，并与此接收端口形成对应关系，以此形成MAC转发表。

一般词汇

TDMA

时分多址（Time division multiple access）是一种为了实现共享传输介质或者网络的通信技术。

目的是避免冲突。

时分多址是把时间分割成周期性的帧(Frame)每一个帧再分割成若干个时隙向基站发送信号，在满足定时和同步的条件下，基站可以分别在各时隙中接收到各移动终端的信号而不混扰。TDM把time slot用在multiplexing，TDMA把time slot用在multiple access。

TDM

时分多路复用（time-division multiplexing）是将不同的信号相互交织在不同的时间段内，沿着同一个信道传输；在接收端再将各个时间段内的信号提取出来还原成原始信号的通信技术。

目的是在同一个信道上传输多路数字信号，且避免碰撞。

由于根据抽样定理得到的抽样脉冲之间有时间空隙，则可以利用这种空隙传输其他信号的抽样值。分为同步和异步时分复用系统。

总线型拓扑

总线型拓扑是一种基于多点连接的拓扑结构，是将网络中的所有的设备通过相应的硬件接口直接连接在共同的传输介质上。

总线布局的特点：结构简单灵活，非常便于扩充；可靠性高，网络响应速度快；设备量少、价格低、安装使用方便；共享资源能力强，便于广播式工作。

总线有一定的负载能力，因此，总线长度有一定限制，一条总线也只能连接一定数量的结点。最著名的总线拓扑结构是以太网(Ethernet)。

干线

干线运输是指总线拓扑中起骨干作用的线路。

总线型拓扑就要这样的一条干线来连接其他设备。

各个节点设备用一根总线（如同轴电缆等）挂接。

计算机通信网络-6-传输层

发表于 2021-04-20 更新于 2021-04-23 分类于 学习印记

传输层概述

  • 为什么需要传输层？
    • 计算机多任务特性：单个计算机多条通信同时管理，每条通信不同的服务需求。
    • 网络层功能不能满足应用的需求。
  • 服务：屏蔽具体网络细节和不同网络差异，为上层提供与具体网络无关的通信服务。
  • 功能：各种网络的适配，网络通信的分流和复用、多个并发通信的管理、流量控制等。
  • 服务质量（QoS）：将服务质量的好与坏等级化、量化。

传输层应对的问题

  1. 多任务：单节点的多任务，一个节点有多个端。
  2. 网络存储化：
    • 路径过长，且存在多条路径，易长延时，易后发先至。
    • 中介节点多采用存储转发技术，可能重复。
    • 网络中很多待传报文，有滞后，造成拥塞。

传输层连接管理

寻址技术

  • 编址：区分+定位
  • 特服号码方案：各种服务预制固定不变的端口号。
    • 浪费端口，浪费服务进程消耗。
    • 特服不够“特”，不能“广而周知”。
  • “前台接待”方案：由前台唤醒服务进程，前台将客户分流到具体服务窗口。
  • “查号台”方案：客户机到第三方查询。

三次握手建立连接

核心目的是双方序号的商定。

  • 解决长延时报文问题。
  • 利用序号差判断生命期。
  • 定原则
    • 连接的起始序号不能从0开始。
    • 新连接需在旧连接结束后T_生命期才能建立。

四次挥手释放连接

  1. 两种释放策略
    • 非对称释放：任意一方均可释放整个连接。
    • 对称释放：每个方向独立释放本方单向连接，不发送仍能接收。
  2. 确认丢失，若超时没有得到确认，拆除连接。
  3. 超时没有收到任何数据，拆除连接。

TCP的流控

  1. 基于接收方缓存状态反馈的流控。
  2. 传输层流控核心问题：往返延时变化大。
  3. RTT的测量——动态往返延时，平滑处理预测。
  4. 动态RTO（超时重传）：通过公式计算得到，保证对RTT变化适度敏感，同时使运算不至于太麻烦。
  5. 两种极端情况的处理：小报文数量消减
    • 发送得慢而少：延迟发送。
    • 接收得慢：发送方受流控会停止，延迟应答。
  6. TCP拥塞控制技巧：
    • 慢启动
    • 拥塞避免：每次增加一个窗口长度
    • 拥塞检测：连续三个重复确认、窗口减半逐个增加……
