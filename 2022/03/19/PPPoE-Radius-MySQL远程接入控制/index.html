PPPoE-Radius-MySQL远程接入控制

发表于 2022-03-19 更新于 2022-08-01 分类于 学习印记 阅读次数：

综合课程设计，第二阶段，配置PPPoE服务器远程接入Radius服务器，并采用sql数据库保存用户信息，实现接入控制。

参考链接：
https://blog.csdn.net/Openking/article/details/42561819

https://www.cnblogs.com/changelee/p/7533469.html

https://www.linuxidc.com/Linux/2015-06/119102.htm

步骤概述

  1. 在对应的机器上安装所需的软件包，拷贝radius客户端出来
  2. radius服务器修改clients.conf，目的是设置与NAS端共享的公钥，以及对应NAS的IP
  3. radius服务器修改 /etc/raddb/users，目的是保存设置用户的账户密码，在不使用sql时就到这个文件夹下查询。
  4. 此时已经可以只使用radius服务器，在本地，通过验证。
  5. 在pppoe服务器上，/etc/ppp/radiuds目录下，添加一行到 servers，目的是填入radius服务器的地址，以及之前在第二步设置的公钥。
  6. 同目录下，修改dictionary文件。
  7. 同目录下，修改radiusclient.conf，这是radius客户端最主要的配置文件，关联了上面的servers文件，设置了radius服务器的ip和端口号。
  8. 修改/etc/ppp/pppoe-server-options，使pppoe服务器与radius客户端相关联。
  9. 至此，已经实现pc-pppoe-radius远程认证。
  10. 将 /etc/raddb/sql/mysql/schema.sql 这个数据库导入mysql数据库中，并添加用户对应信息
  11. 修改 etc/raddb/site_enabled/default，配置验证记账有关信息。
  12. 修改 /usr/local/etc/raddb/sql.conf ，填入mysql的账户密码，使radius与数据库相连。
  13. /ect/raddb/radius.conf ，去掉$INCLUDE sql.conf的注释，应该是表示radius调用sql.conf
  14. 参考拓扑：

  3. 启动服务器服务 radiused -X /usr/sbin/radiusd -X

    • 在Radius服务器本机中使用命令 radtest test test localhost 0 testing123

      第一个test为用户名，第二个test为密码，localhost表示要访问的服务器的地址，要去的端口为0，共享密钥为testing123。这个连接没有成功，证明之前配置的test有误。
    • 使用命令radtest kc 123 localhost 0 testing123

      证明用户面加双引号的配置是正确的。

    可以在另一台主机使用命令radtest kc 123 192.168.11.11 0 testing123-1正确连接
